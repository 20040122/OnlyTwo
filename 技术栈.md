  路由层

  - app/layout.tsx
    全局 HTML、字体、全局 provider 挂载点。
  - app/page.tsx
    首页入口。已登录跳聊天，未登录跳落地页或登录页。
  - app/(auth)/login/page.tsx
    登录表单，只负责 UI 和提交 action。
  - app/(auth)/signup/page.tsx
    注册表单，只负责 UI 和提交 action。
  - app/(auth)/callback/route.ts
    处理 Supabase Auth 回调，换 session，重定向到 onboarding 或 chat。
  - app/(setup)/onboarding/page.tsx
    完善 profiles，比如昵称和头像。
  - app/(setup)/invite/page.tsx
    展示邀请码、邀请链接、当前关系状态。
  - app/(setup)/accept/[code]/page.tsx
    接受邀请页，读取邀请码并触发绑定。
  - app/(chat)/layout.tsx
    聊天主壳，做鉴权检查、资料检查、绑定状态检查。
  - app/(chat)/page.tsx
    聊天区入口页，通常直接跳唯一会话页或渲染空状态。
  - app/(chat)/chat/page.tsx
    主聊天页，组装消息流、输入框、已读逻辑。
  - app/(chat)/settings/page.tsx
    账号、关系、解绑等设置。

  API 层

  - app/api/relationship/invite/route.ts
    创建或查询有效邀请码。
  - app/api/relationship/accept/route.ts
    接受邀请码，创建 relationships、conversations、conversation_members。
  - app/api/messages/route.ts
    拉消息和发消息。首期也可以只保留 POST。
  - app/api/messages/read/route.ts
    更新 conversation_members.last_read_message_id 和 last_read_at。

  业务域

  - features/auth/actions.ts
    登录、注册、退出登录。
  - features/auth/server.ts
    读取当前用户 session、获取当前 user。
  - features/auth/components/login-form.tsx
    登录表单组件。
  - features/auth/components/signup-form.tsx
    注册表单组件。
  - features/profile/actions.ts
    创建或更新 profiles。
  - features/profile/server.ts
    查询当前用户 profile、判断 onboarding 是否完成。
  - features/profile/schemas.ts
    昵称、头像等表单校验。
  - features/relationship/actions.ts
    生成邀请码、接受邀请、解绑关系。
  - features/relationship/server.ts
    查询当前关系、查询邀请码、校验是否已绑定。
  - features/relationship/guards.ts
    收口业务约束：一个用户只能绑定一次、不能给自己绑、只有双方可访问。
  - features/relationship/types.ts
    relationship_invites、relationships 的领域类型。
  - features/conversation/server.ts
    通过 relationship_id 查唯一 conversation，查会话成员。
  - features/conversation/queries.ts
    会话查询拼装，给页面直接消费。
  - features/message/actions.ts
    发送消息、标记已读、失败重试。
  - features/message/server.ts
    查消息列表、做消息权限检查。
  - features/message/components/message-list.tsx
    消息流展示。
  - features/message/components/message-input.tsx
    左右气泡、状态、时间。
  - features/message/types.ts
    浏览器端 Supabase client。
    服务端 Supabase client。
  - lib/env.ts
  - relationship 和 conversation 必须分开，虽然当前是一对一，但一个管关系真相，一个管聊天真相。
  - “发送中/失败”是前端状态，不要一上来就全写进数据库。
  - presence 先独立目录，但首期不要建表，直接走 Realtime。
