  请使用 React + Tailwind CSS 实现一个高保真产品落地页 / 登录页 / 聊天页面，服务于一个 1v1 亲密关系聊天产品 “Only Two”。

  设计目标：
  页面气质要高级、轻盈、克制、通透，参考 Apple 官网式的现代审美，但不要照搬 Apple。整体要有情绪感和亲密关系感，不要做成传统社交产品后台，也不要做成俗套的情侣
  App。

  视觉风格要求：
  - 使用浅色系流动渐变背景，颜色以粉色、奶油白、浅橙、淡金、浅蓝、淡紫为主
  - 背景不是纯色，要有多层柔和渐变和模糊光斑
  - 页面主体容器使用半透明白色 + backdrop blur，形成轻玻璃拟态效果
  - 整体低饱和、高亮度、留白充足
  - 大圆角、轻阴影、柔和高光，不要硬边框和重描边
  - 局部可以有高级感 3D 效果，但不要浮夸

  组件要求：
  - 按钮使用胶囊型大圆角按钮
  - 主按钮要有持续呼吸式发光效果，用 box-shadow + animation 实现
  - hover 时按钮光晕增强、颜色轻微变化，并有一点点上浮
  - 卡片可使用 3D 翻转效果，正面展示摘要，背面展示简介
  - 翻转使用 CSS perspective + transform + backface-visibility
  - 卡片背面颜色可比正面更丰富，但仍需保持低饱和高级感
  - 图标尽量使用简洁 SVG，不依赖厚重插画

  布局要求：
  - 使用单屏或接近单屏的居中布局
  - 结构优先采用：顶部品牌符号 / 中部大标题 / CTA 按钮 / 下方特性卡片
  - 标题要醒目但简洁，副标题不要太长
  - 信息密度低，视觉重心明确
  - 桌面端和移动端都要自然，不允许桌面好看移动端崩掉

  品牌视觉要求：
  - 可以使用“双人关系”的抽象符号，例如两颗彼此分开的心、两个悬浮圆形、两个渐变图形
  - 表达要克制和高级，不要使用俗套连线、玫瑰、满屏爱心、婚庆感元素
  - 更强调“两个独立个体之间的专属连接感”

  动效要求：
  - 背景做缓慢流动渐变，用 background-size + keyframes 实现
  - 按钮做柔和呼吸光效
  - 卡片翻转要顺滑，速度偏慢
  - 所有动效都要克制、柔和、持续存在
  - 避免 bounce、闪烁、过快位移、廉价粒子效果

  代码要求：
  - 使用 React 函数组件
  - 使用 Tailwind CSS 编写主要样式
  - 少量复杂动画可写在 globals.css 中，用自定义 keyframes 和 class
  - 代码结构清晰，不要把所有样式写得混乱难维护
  - 保持语义化 JSX 结构
  - 优先输出可以直接落地运行的页面代码
  - 不要引入不必要的第三方动画库，优先使用 CSS 实现
  - 如果使用 SVG 图标，直接内联
  - 注意 hover、transition、阴影、透明度、blur 的组合质感

  避免事项：
  - 不要使用默认模板感很强的布局
  - 不要使用紫色主导的 Dribbble 风
  - 不要使用深色赛博朋克风
  - 不要使用复杂 dashboard 风格
  - 不要让页面显得拥挤或信息过载
  - 不要出现廉价恋爱 App 视觉语言

  输出要求：
  - 直接输出完整的 React + Tailwind 页面代码
  - 如果需要补充动画，请同时给出对应的 globals.css 代码
  - 页面应具备完整视觉层次，而不是只有文字和按钮
